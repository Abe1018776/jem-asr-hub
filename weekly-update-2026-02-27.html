Confidence Scoring — Addressing Your Feedback

What You Asked For

"Instead of seeing the timecode, can we see the actual confidence rating — the TC isn't so important if it synced anyway."

You also asked about the orange and red colors in the UI. Here's the full picture:

What We Found

The confidence data is already flowing through the entire pipeline. The alignment engine (based on ivrit-ai's breakable aligner, adapted for Yiddish) computes a w.probability score for every single word during alignment. This score (0.0–1.0) represents how confident the Whisper model is that a given word matches the audio at that position. The API already returns these scores in the JSON response.

The UI already uses these scores for color-coding the word chip borders (green, yellow, red), but currently displays the timecode (e.g., "3.45s") as the text below each word instead of the confidence number. That's the gap we're closing.

About Orange vs Red

The orange/red you noticed comes from two different places:

  • ■ Red in the document cleaning UI = removed text (strikethrough). This is binary: removed vs kept, not a confidence level.
  • ■ Orange/Amber in the alignment UI = moderate confidence word (0.2–0.5 probability). These words synced but the model isn't fully certain.

With the updated UI, users will see the exact confidence number alongside the color, making it immediately clear which words to trust and which to review.

End-to-End User Journey

Here is the complete path from raw audio to benchmarked, production-ready ASR output. The first three phases are operational today; phases 4 and 5 are in active development.

Phase 1: Map Audio to Transcripts Live

Go to mapping.kohnai.ai. Browse 4,669 audio files and 1,065 transcripts. The system automatically suggests matches using date + keyword analysis with confidence scores. Review suggestions, verify matches, and export the verified pairs as CSV. The curated 50-Hour selection tab has 423 pre-matched pairs ready for training.

Phase 2: Align Audio with Text Live

Go to align.kohnai.ai. Upload an audio file and paste the corresponding Yiddish text. The system generates word-level timestamps using the yi-whisper model on GPU. Review results with karaoke-style playback — click any word to hear it. Each word is color-coded by confidence (green/yellow/red). Export timestamps for downstream use.

Phase 3: Clean & Prepare Transcripts Live

In the Yiddish Cleaner app, raw transcripts pass through an 8-stage cleaning pipeline. Stage directions, brackets, editorial notes, and formatting are automatically stripped. Choose from rule-based presets (minimal, full, heavy) or LLM-powered cleaning via 5 different AI providers. Each cleaning gets a quality score (0–100) so you know how confident the removal decisions were.

Phase 4: Benchmark ASR Accuracy Building

Run transcriptions through multiple ASR models and measure accuracy against gold standard references using WER/CER metrics with Yiddish-specific Hebrew normalization. The public leaderboard ranks models across aggregate scores, per-file breakdowns, and side-by-side comparisons. Currently 5 gold standard files; expanding to 7.

Phase 5: Review, Train & Publish Planned

The JEM review interface will combine karaoke playback with per-word confidence display and approve/reject controls. Reviewed data feeds into training pipelines with strict separation from benchmark audio. Fine-tuned models are tested against the benchmark and published when they meet quality thresholds.
